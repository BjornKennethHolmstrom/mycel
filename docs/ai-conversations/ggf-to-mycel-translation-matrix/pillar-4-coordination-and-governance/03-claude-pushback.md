Yes, I'll push back here, and your instinct is exactly right.

This pillar has built an entire digital governance operating system — proposals with 14-day deliberation timelines, conflict resolution event kinds, values diagnostics tools, delegation protocols, cognitive friction audits, System Canary roles. It's a digital parliament. That's the opposite of what Mycel should be.

**The fundamental problem:** This design assumes people govern *through* Mycel. But Mycel's whole philosophy is that it gets out of the way. It helps people find each other, then they go talk. The five neighbors coordinating storm response don't need a formal proposal event with a consent-check phase and values mapping. They need to know who's around and then walk over.

Gemini's additions make it even heavier — Dunbar's number enforcement, automatic "Capture Warnings," subsidiarity forks, cryptographic TTLs on guardianship. This is governance theory encoded as software. It's fascinating on paper and completely unusable for 5-50 people sharing tools.

**What's actually useful here, stripped to the bone:**

- Constellations (temporary groups that form around a need and dissolve). You already have this from the technical sketch. It's the only governance primitive Phase 0 needs.
- The principle that governance is emergent, not declared. Don't build governance tools — let people self-organize using presence, messaging, and constellations.

Everything else — BAZ overlays, proposals, conflict events, delegation, audits, guardianship, non-participation statements — is designing institutions before a community exists to inhabit them. If 50 people are using Mycel and they need a proposal system, they'll ask for one, and they'll know what shape it should take because they've been actually coordinating.

**The deeper point your instinct is touching:** Mycel should create *reasons to meet in person*, not substitute for meeting in person. "I can see my neighbor has tools to offer" → I walk over. "Three of us are paying attention to water safety" → we meet at someone's kitchen table. The ambient map is a catalyst for physical coordination, not a replacement for it. Every feature that pulls deliberation *into* the app works against this.

So for Phase 0, the governance pillar is one line: **constellations with expiry**. That's it.
